# Agent OnBoard — Work Server

Local execution server that runs whitelisted commands on your machine on behalf of Agent OnBoard sessions.

## Quick Start

```bash
cd work-server
chmod +x setup.sh
./setup.sh
```

This will:
1. Create `~/agent-onboard-scripts` directory
2. Install npm dependencies
3. Generate `.env` with a random auth token

## Configuration

Edit `.env` after setup:

| Variable | Description |
|---|---|
| `WORK_SERVER_TOKEN` | Auth token (auto-generated by setup) |
| `SUPABASE_URL` | Your Supabase project URL |
| `SUPABASE_ANON_KEY` | Your Supabase anon/public key |
| `SCRIPTS_DIR` | Directory for approved scripts (default: `~/agent-onboard-scripts`) |
| `PORT` | Server port (default: `3002`) |

## Running

```bash
npm start        # production
npm run dev      # development with auto-reload
```

## API Endpoints

All endpoints except `/health` require the `x-work-token` header.

| Method | Path | Description |
|---|---|---|
| GET | `/health` | Server status and whitelist |
| POST | `/execute` | Run a whitelisted command |
| POST | `/read-file` | Read a file from scripts dir |
| POST | `/write-file` | Write a file to scripts dir |
| POST | `/list-files` | List files in scripts dir |

### POST /execute

```json
{
  "command": "node hello.js",
  "sessionId": "uuid",
  "agentId": "uuid",
  "timeout": 30000
}
```

### Whitelisted Commands

`python`, `python3`, `node`, `npm`, `npx`, `echo`, `ls`, `pwd`, `cat`, `grep`, `curl`, `openclaw`

## Security

- Token-based authentication on every request
- Command whitelist prevents arbitrary execution
- Shell operators (`; | & $ \``) are blocked
- File operations restricted to scripts directory
- Execution timeout (default 30s, max 60s)
